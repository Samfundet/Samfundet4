# Serving React through Django

Having samfundet.no appear in search engines with proper titles, descriptions
and images is an absolute requirement for us. A lot of users use Google to find
events and specific info pages. This introduces some complexity, since we can no
longer just build our frontend React app and serve it as static files, since
then all that Google's indexer sees is the content in `index.html`. We
definitely don't want only "You need to enable JavaScript to run this app"
appearing in search results. Additionally, we miss out on proper HTTP response
codes. If we request `samfundet.no/events/blablabla` we should expect to get a
HTTP 404, but instead get HTTP 200.

Normally, people solve this by adding SSR (server side rendering) to the React
project, but this requires another service running on the server in addition to
our Django backend, which ITK wasn't too keen on.

Instead, we solve this by having Django serve the React frontend. All requests
to Samf4 hit our Django backend, which has knowledge over all frontend routes 
(see [Export frontend routes](#export-frontend-routes)), and based on the
requested page, populates `index.html` with the relevant metadata and sets the
correct HTTP status code.

## Routing

The Django backend serves both its own API routes *and* the frontend routes.
The URL layout (`backend/root/urls.py`) looks like this:

| Path            | Serves                                             |
|-----------------|----------------------------------------------------|
| `/api/...`      | All backend API routes (`samfundet/urls.py`)       |
| `/admin/...`    | Django admin                                       |
| `/assets/...`   | The built frontend bundle (JS, CSS, fonts, images) |
| `/media/...`    | User-uploaded files                                |
| everything else | The React frontend (`samfundet/routing/urls.py`)   |

All API routes live under the `/api/` prefix so they never collide with
frontend routes. E.g. `/api/login/` is the login endpoint, while
`/login/` is the login page.

The frontend routes are generated automatically: `samfundet/routing/urls.py`
creates a Django urlpattern for every route in the autogenerated
`frontend_routes.py` (see [Export frontend routes](#export-frontend-routes)).
Each route is served by `react_view`, which simply returns the built
`index.html`. A route can be given a custom view (e.g. to inject metadata or
return a different status code) by adding it to `ROUTE_VIEW_OVERRIDES`.

A request which matches no route at all hits a catch-all which returns the
React app with a HTTP 404 status code, and the frontend then displays the Not
Found page.

Note: `/assets/` is not a urlpattern: it is served by
[WhiteNoise](https://whitenoise.readthedocs.io/) middleware, which handles
requests for static files (collected into `staticroot/` by `collectstatic`)
before they reach URL routing.

## Export frontend routes

In order for Django to serve the frontend routes (and add metadata to e.g. the
`/events/{id}` page), it needs to know their paths.

We achieve this by running a frontend script which precompiles all frontend
routes into a format which Django understands
(`backend/samfundet/routing/frontend_routes.py`). Run `yarn export-routes` in
the frontend directory to regenerate it. New frontend routes are picked up and
served automatically by Django.

The reverse also exists: `python manage.py generate_routes` exports all backend
routes to `frontend/src/routes/backend.ts`.

## Building React app

To precompile the frontend routes, and build the React app into the location
Django expects, run the `build_to_backend.sh` script.

## Setting metadata

Look at the `backend/samfundet/routing/metadata.py` file.

Example:

```python
def react_event_view(request: HttpRequest, **kwargs: Any) -> HttpResponse:
    try:
        event = Event.objects.get(id=kwargs['id'])
    except Event.DoesNotExist:
        # ...

    title = f'{event.title_nb} - Samfundet'
    description = event.description_short_nb

    metadata = Metadata(title=title, description=description)

    # Add custom metadata items
    metadata.items.append(MetadataItem('name', 'og:custom', 'Lorem ipsum'))

    html = get_frontend_html()
    return HttpResponse(inject_metadata(html, metadata))
```

Then register the view in `ROUTE_VIEW_OVERRIDES`
in `backend/samfundet/routing/urls.py`:

```python
ROUTE_VIEW_OVERRIDES: dict[str, Callable[..., HttpResponse]] = {
    'EVENT': react_event_view,
}
```
